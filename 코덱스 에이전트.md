# 코덱스 에이전트 운영 메모

이 문서는 이 저장소에서 Codex가 긴 대화나 작업 전환 시 상태를 유지하기 위한 로컬 메모다.

## 운영 규칙

- 대화가 길어지거나 작업 컨텍스트가 커지면 이 문서를 우선 갱신한다.
- 아래 3개 섹션만 유지하고, 불필요한 설명은 줄인다.
- 새로운 결정이 생기면 `결정된 것`에 추가한다.
- 아직 확정되지 않은 이슈는 `안정해지지 않은 것`에 남긴다.
- 바로 이어서 실행할 일은 `다음 할 일`에 최신 상태로 적는다.

## 결정된 것

- 로컬 작업 저장소 경로는 `C:\codex\codex-medipic-app`이다.
- 현재 저장소는 Git 상태가 깨끗한 작업용 복사본으로 정리되어 있다.
- 이 저장소는 Next.js 16 기반 앱이며, 루트 규칙은 `AGENTS.md`를 따른다.
- 긴 대화나 작업 전환 시 이 문서를 로컬 작업 메모처럼 사용한다.
- 운영 관련 지식과 공통 데이터는 `G:` 드라이브를 우선 사용하고, 개발 저장소와 실행 파일은 각 PC 로컬에 둔다.
- 사용자는 3대 PC를 오가며 작업하므로, Codex는 PC 이동을 전제로 최신 운영 메모와 작업 상태를 먼저 대조해야 한다.
- `G:\내 드라이브\codex\LLM Wiki.md`는 LLM Wiki의 상위 개념 문서다.
- 실제 운영 위키는 `G:\내 드라이브\OBSIDIAN-SECOND\wiki\`에 있으며 `overview.md`, `index.md`, `log.md` 구조를 사용한다.
- `G:\내 드라이브\codex\CODEX_OPERATING_RULES.md`, `CODEX_MASTER_STATUS.md`, `projects\codex-medipic-app.md`를 중앙 handoff 기준으로 사용한다.
- `src/hooks/useLocale.ts` lint 오류는 수정 완료했다.
- 관리자 인증은 `.env` 기반 `DOCTOR_PASSWORD`, `DOCTOR_SESSION_SECRET`를 쓰도록 변경했다.
- `.env.example`를 추가했다.

## 안정해지지 않은 것

- `npm run lint`는 `src/hooks/useLocale.ts`의 effect 내부 `setState` 문제로 실패한다.
- 관리자 인증은 `DOCTOR_PASSWORD` 기본값과 고정 쿠키 토큰 사용으로 보안 보완이 필요하다.
- `vercel.json`의 `/uploads/(.*)` rewrite 대상이 실제 route와 맞는지 재검토가 필요하다.
- 프로덕션 저장소가 현재 로컬 JSON / in-memory fallback 구조라 실제 운영용 DB 설계가 미완료 상태다.
- Codex 운영 규칙이 여러 문서로 분산되어 있다: 저장소 `AGENTS.md`, `G:\내 드라이브\AI개발계획\codex\JH Codex Global Development Operating Guide.md`, `Project AGENTS.md`, `OBSIDIAN-SECOND\wiki` 문서.
- 일부 `G:` 문서는 현재 세션에서 한글 인코딩이 깨져 보여 재사용성과 자동 처리 안정성이 떨어진다.
- `LLM Wiki.md`의 추상 개념 문서와 `OBSIDIAN-SECOND\wiki`의 실제 운영 위키 사이에 역할 구분은 있으나 연결 규칙이 명시적으로 정리돼 있지 않다.
- PC를 바꿔 접속할 때 Codex가 항상 동일한 최신 상태를 읽도록 하는 중앙 handoff 문서가 아직 없다.
- `npm run lint`는 이제 오류는 없지만 warning 11개가 남아 있다.
